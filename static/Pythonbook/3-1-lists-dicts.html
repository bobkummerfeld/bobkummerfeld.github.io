3.1 Lists and Dictionaries¶

Python Containers: Lists¶

Python has a data structure that lets you store a sequence of objects. It’s called the “list”.


 

 

 

 

 

 

A list: command line arguments¶

When you start a program by typing the name on the command line, the arguments you put on the line are passed into your program as a list of strings. Watch this to see how.


 

 

 

 

 

 

Python Tuples¶

As well as lists, Python has a container data structure called a “tuple”. The difference between and a list is the tuple contents cannot be changed: it is “immutable”.


 

 

 

 

 

 

Python List Slices¶

As well as accessing a single element of a list using an index, you can also access a sub-section of the list called a “slice”.


 

 

 

 

 

 

Dictionaries!¶

The dictionary data structure is the most important for Python. It looks like a list, but instead of indexing with a number, you can use any immutable object to do the lookup.


 

 

 

 

 

 

Mutable and Immutable Data¶

Most data structures in Python are “mutable”. This means they are changeable. You must be a little careful with mutable data structures because you might accidentally modify them. But, there is also an “immutable” data structure in the form of a “tuple”. A tuple is the same as a list except not changeable.
